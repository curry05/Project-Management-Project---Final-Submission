
SDLC Project Management – Super Short Notes
SDLC Overview: - Structured software process → better planning, quality,
communication.
1.  Planning:
-   Define problem, goals, scope.
-   Gather requirements.
-   Feasibility + time/cost estimates.
-   Project & requirements docs.
2.  Design:
-   System architecture.
-   UI mockups.
-   Data models (ERD/UML).
-   Module breakdown + specs.
3.  Implementation:
-   Coding + integration.
-   Follow standards, use Git.
-   Developer testing.
4.  Testing:
-   Unit, integration, system, UAT.
-   Debug + defect tracking.
5.  Deployment:
-   Install system, configure, migrate data.
-   Training + release.
6.  Maintenance:
-   Bug fixes, updates, performance & compatibility improvements.
Models: - Waterfall, Iterative, Agile, Spiral.
Tools: - Gantt, PERT, Kanban, Git/GitHub.
Roles: - PM, Analyst, Architect, Developer, Tester, User.
